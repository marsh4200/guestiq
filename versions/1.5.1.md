## v1.5.1 — Automation back end no longer named in the console

The underlying automation platform is never mentioned in anything the operator
or the client sees. The Automation tab now reads **"AR Smart automation"**.

- Every label, hint, status message and error in the Automation tab reworded:
  "Hub URL", "Access token", "Enable automation sync", "Room name prefix sent
  to the hub". The token hint no longer walks through a third-party UI.
- Connection test and sync results reworded — *"Cannot reach AR Smart
  automation"*, *"Token rejected (401) — generate a new access token on the
  hub"*, *"Connected — synced 6 room(s) to AR Smart automation"*.
- Checkout dialog and the overdue-alerts settings card no longer name the
  platform either.
- Staff permission renamed to **Automation & smart control**.
- Historical changelogs shown in the Updates tab were scrubbed too, so an
  older install updating forward never sees the name.
- The label is configurable with the `GUESTIQ_AUTOMATION_NAME` environment
  variable (defaults to `AR Smart automation`) and is served to the console
  from `GET /api/automation`, so nothing is hard-coded in the browser.

Wiring is unchanged: same REST service call, same webhook fallback, same
`set_occupancy` / `persistent_notification.create` endpoints, same settings.
