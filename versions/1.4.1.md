## v1.4.1 — Update source hidden behind the AR Smart brand

- The Updates tab no longer names GitHub or shows the repository and branch.
  It now reads **"Latest on AR Smart server"**, and the repo / branch fields
  were removed from the `/api/update/check` response entirely — the browser is
  never told where the code actually comes from.
- The label is configurable with the `GUESTIQ_UPDATE_SOURCE` environment
  variable (defaults to `AR Smart server`); the real origin still comes from
  `GUESTIQ_REPO` / `GUESTIQ_BRANCH`, server-side only.
- Update progress screen reworded: "host watcher" → "update service",
  "pulling" → "downloading". Failure messages point at the update service on
  this server instead of naming git or the repo.
- Failed update check now shows a Retry button instead of a dead end.
