## v1.3.0 — UI overhaul & live update progress

### Updates
- **Live update progress screen** — "Update now" opens a full-screen progress
  view with a real progress bar, step tracker and live log:
  queue → host watcher pickup → pull & rebuild → back online → auto-reload.
- New `/api/update/status` endpoint so the UI can watch the watcher consume
  the update flag, then follows `/api/health` for the version change.
- Detects a watcher that never picks up the request (3 min) and a rebuild
  that stalls (5 min), with the manual fallback command shown in the log.
- Reloads the console automatically 3 s after the new version is confirmed.

### Admin UI
- **Proper navigation** — icon pill nav (sticky, horizontally scrollable on
  mobile) replaces the old flat tab strip; pending check-in count badge on
  the Check-ins tab and an update badge on Updates.
- Redesigned topbar with logo mark, pulsing update indicator and sign-out icon.
- **Real checkout modal** — replaces the browser confirm() popup: shows guest,
  room, check-in / due-out summary and what happens on checkout (room freed,
  HA automations to unoccupied).
- All browser confirm() popups replaced with styled confirmation modals
  (cancel check-in, delete room, delete guest, start update).
- Modals gained headers with icons and a close button, plus open animations.
- Stat cards on the Check-ins tab: pending arrivals, guests in-house,
  rooms available, total rooms.
- General polish: input focus rings, button hover/press states, animated
  toasts, refined tables, new login screen with logo.
