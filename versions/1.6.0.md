## v1.6.0 — Property logo on the guest pages

Upload the hotel / lodge logo once and it appears, with the property name, on
every page a guest reaches by scanning a QR code.

### Where it shows
- **Room QR page** — above the property name, including the locked
  "stay ended" version.
- **Arrival check-in QR page** — above the welcome message.
- Falls back to the initials of the property name when no logo is set, or if
  the image fails to load.

### Uploading
**Settings → Hotel details → Logo.** Pick a file and it uploads immediately,
with a live preview and a Remove button. PNG, JPG, WEBP, GIF or SVG, up to
3 MB; a square logo on a transparent background looks best.

- Only one logo is ever kept — a new upload replaces the old file.
- Stored in `data/branding/`, which is bind-mounted, so it survives container
  rebuilds and updates.
- Requires the *Change hotel settings* permission, so staff can't replace the
  branding.
- Served from `GET /api/logo` (public — guest pages have no login) with a
  cache-busting version so a replaced logo shows up immediately rather than
  after the browser cache expires.
- Uploaded SVGs are served with `nosniff` and a locked-down
  `Content-Security-Policy`, so a logo file can never execute anything.

### API
- `POST /api/settings/logo` (multipart) · `DELETE /api/settings/logo` — needs
  the settings permission
- `GET /api/logo` — public image
- `logo_url` added to `GET /api/public/branding`, `GET /api/room/{code}` and
  `GET /api/settings`
