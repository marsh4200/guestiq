# GuestIQ 1.2.0 — Long-lived token auth

- Automation tab now takes a Home Assistant **long-lived access token**
  (Profile → Security → Long-lived access tokens → Create token).
- With a token, GuestIQ calls HA's authenticated REST API directly
  (ar_smart_loadmanager.set_occupancy service) — no webhook ID needed.
- Webhook mode kept as fallback when no token is entered.
- Test connection now verifies the token against /api/ and reports
  401s clearly before attempting a sync.
